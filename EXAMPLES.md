### 2. メッセージ処理の流れ

**ステップ1: メッセージ取得**
```
[新しいメッセージを取得] ボタンをクリック
↓
サンプルメッセージが生成される:
- "この場所に荷物おいていいですか？" (booking.com)
- "予約どれぐらい開いていますか？" (booking.com)  
- "この周辺におすすめの観光地はございますか？" (booking.com)
```

**ステップ2: 返信候補生成**
```
[返信候補を取得] ボタンをクリック
↓
3つの候補が自動生成される:

候補1: "お荷物の預かりサービスをご利用いただけます。フロントデスクまでお越しください。"
信頼度: 0.9 | タイプ: informative

候補2: "お荷物をお預かりいたします。安全に保管いたしますのでご安心ください。"
信頼度: 0.8 | タイプ: reassuring

候補3: "承知いたしました。お荷物の預かりは可能です。お部屋の準備ができ次第、お荷物をお部屋までお運びいたします。"
信頼度: 0.7 | タイプ: service_oriented
```

**ステップ3: 返信送信**
```
[この候補で返信] ボタンをクリック
↓
✅ 返信を送信しました！
```

## 🔧 コマンド例

### Docker Compose操作
```bash
# サービス起動
docker-compose up -d

# ログ確認
docker-compose logs -f api

# サービス停止
docker-compose down

# データベースリセット
docker-compose down -v
docker-compose up -d
```

### 手動実行
```bash
# FastAPIサーバー起動
python -m uvicorn app.main:app --reload

# Streamlit起動
streamlit run streamlit_app.py

# データベース初期化
python -c "from app.database import create_tables; create_tables()"

# サンプルデータ投入
python app/seed_data.py
```

### APIテスト
```bash
# ヘルスチェック
curl http://localhost:8000/health

# ホテル一覧取得
curl http://localhost:8000/hotels

# メッセージ取得
curl http://localhost:8000/messages/1
```
